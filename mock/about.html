暮らしの情報は、
バラバラに点在している

既存の行政GISと市民の間には、大きな距離があります。 私たちは4つの課題を解決するために、このサービスを立ち上げました。

01
build_circle

既存GISは業務向けUI

「たみまっぷ」は業務利用を想定して作られており、 情報の階層が深く、スマホ最適化も不十分。 市民が知りたい情報にたどり着きにくい設計です。

02
picture_as_pdf

ゴミ出しルールがPDFのみ

ゴミ収集ガイドブックは60ページを超えるPDF。 検索もできず、スマホでは読みにくい。 「これって何ゴミ？」という日常の疑問に応えられていません。

03
translate

外国人住民の情報アクセス

伊丹市には5,000人以上の外国籍住民が暮らしています。 日本語中心のPDFや複雑なサイトは、 生活情報への大きなハードルになっています。

04
account_tree

データのサイロ化

オープンデータ、都市計画、防災、廃棄物 —— それぞれが別の担当部署・別のシステムで管理され、 市民からは「1つのサービス」として使えません。

lightbulbソリューション

私たちのアプローチ

統合・モバイル最適化・オープンソース —— 3つの方針で、市民にとっての「使える地図」を実現します。

hub

統合

6種類以上の行政データをひとつの地図上に束ね、レイヤ切替で自由に組み合わせ可能。

smartphone

モバイルファースト

スマートフォンで片手操作できる UI を徹底追求。Material Design 3 準拠で直感的な操作感。

code

OSS 活用

MapLibre GL JS / PMTiles など OSS スタックで構築。ライセンスコストゼロ、永続的な運用が可能。

offline_bolt

オフライン対応

PWA として動作し、災害時や電波の届かない場所でも、地図とゴミ出し情報を閲覧できます。

stack技術的特徴

モダンなOSSスタック

枯れた OSS と最新のフロントエンド技術を組み合わせ、低コストかつ高品質な体験を実現しています。

  1. palette
    UI

    Material Design 3

    Google の最新デザインシステム。トークンベースのテーマ設計で、ダークモードやブランドカラー変更に柔軟に対応。

  2. map
    Map Engine

    MapLibre GL JS

    Mapbox GL JS の OSS フォーク。高速なベクタータイル描画と、トークン不要の運用が魅力。

  3. database
    Tile Format

    PMTiles (Protomaps)

    単一ファイルで配信できる次世代ベクタータイル形式。静的ホスティングのみで大容量データを高速配信。

  4. public
    Base Map

    国土地理院ベクトルタイル

    日本の公的機関が提供する無料ベースマップ。精度・信頼性ともに商用地図と遜色なし。

  5. workspace_premium
    License

    フルOSSスタック

    ライセンス料ゼロ。既存の ArcGIS 運用と比較して年間数百万円規模のコスト削減が可能。

  6. cloud_off
    PWA

    オフラインファースト

    Service Worker で地図タイルと主要データをキャッシュ。災害時にも確実に機能する防災ツールとして設計。
