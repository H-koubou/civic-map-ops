# Civic Map Ops（旧: itami-civic-map）

このファイルはClaude Codeが本プロジェクトで作業する際のコンテキストです。
セッション復帰時に最初に読む想定。

## ⚠ 重要: プロジェクト方針を大きく転換（2026-04）
当初は「市民向けGIS」として構想していたが、以下の理由で
**業務向け現場記録ツール（SaaS）** に方針転換した。

### 方針転換の理由
- 市民向け情報地図は広告/集客モデルしか成り立たず、自治体予算化の道筋が弱い
- 一方、現場の点検・補修業務は**紙/Excel運用**で非効率なところが多く、
  委託業者・職員の両方にとって「明日から使える」ツールとして刺さる
- SaaSとして有料導入できる説得力が高い
- 伊丹市だけでなく全国の中規模自治体に**横展開前提**で設計することで、
  1自治体あたりの開発コストを薄められる

### 現在のプロダクト定義
- **プロダクト名**: Civic Map Ops（汎用ブランド）
- **テナント表示**: 「伊丹市 環境インフラ台帳」のように差し替え可能
- **ターゲットユーザー**:
  - 自治体職員（環境クリーンセンター／上下水道局）
  - 委託業者（清掃・点検・補修の受託事業者）
- **用途**:
  - タブレットを持って現場に出て、写真＋チェックリスト＋位置情報で記録
  - 紙/Excel運用の置き換え
  - 地図UIで「どの案件が・どこに・どの状態か」を一目で把握
- **差別化軸**: **地図UXが圧倒的に優れていること**（競合はArcGISベースで重い/モバイル弱い）
- **横展開前提**: マルチテナントで他自治体にも展開可能な設計必須

## プロジェクトの目的
自治体のインフラ維持管理（側溝清掃・下水道点検・施設点検等）の
現場業務をタブレット1台で回せるようにし、
紙/Excel運用からの脱却を支援する。

### 扱うデータ
- 廃棄物処理施設（環境クリーンセンター、リサイクル施設）
- 側溝（道路側溝・排水路） ← 主力業務対象
- 下水道管（汚水管・合流管・雨水管） ← 主力業務対象
- マンホール・グレーチング等の附属施設
- 点検履歴・補修履歴・案件（ケース）管理
- 写真・GPS・メモ・チェックリスト・測定値

参考: 市民向けの「ゴミ収集ルール」「分別ガイド」等は**アーカイブ**し、
現時点ではスコープ外とした。

   - URL: https://itami-map.alandis.jp/portal/apps/webappviewer/index.html?id=fbc8e9d00a8d4ee08cccabdc10ef6c00
   - 基盤: ArcGIS WebAppViewer（株式会社アルプスマッピング系？）

3. **G空間情報センター 伊丹市オープンデータ**
   - URL: https://www.geospatial.jp/ckan/organization/hyogo-itami
   - データセット数: 61件（2026-04時点）
   - 形式: ほぼCSV、Shapefileは「地番参考図」のみ
   - ライセンス: CC BY 4.0

4. **ゴミ収集ガイドブック**
   - URL: https://www.city.itami.lg.jp/material/files/group/26/gaidobook20240606.pdf

## 差別化戦略
既存のArcGIS製「たみまっぷ」に対する我々の優位性:

| 観点 | たみまっぷ（既存） | Civic Map Ops |
|------|------------------|--------------|
| ターゲット | 業務閲覧者 | **現場業務オペレーター**（職員・委託業者） |
| 利用シーン | デスクトップで閲覧 | **タブレットで現場記録** |
| デザイン | Esri標準UI | Material Design 3・大きなタッチターゲット |
| データ記録 | 不可（閲覧のみ） | 写真・GPS・チェックリスト・メモ一括記録 |
| 案件管理 | なし | ケース単位で状態管理（未着手/進行中/完了/要再訪） |
| オフライン | 不可 | PWA対応（電波の弱い現場でも記録継続） |
| マルチテナント | 単一自治体固定 | **テナント切替で他自治体展開可能** |
| ライセンスコスト | ArcGIS年間ライセンス必須 | OSSスタックでゼロ |

## 技術スタック（決定版）

### フロントエンド
- **Next.js 15** (App Router) + **TypeScript**
- **MapLibre GL JS** (Mapbox GL JSのOSSフォーク、トークン不要、ベクタータイル対応)
- **shadcn/ui** + **Tailwind CSS v4**
- **PWA**（next-pwa）でオフライン対応

### 地図タイル / ベースマップ
- **国土地理院ベクトルタイル**（無料・公式）
  - https://maps.gsi.go.jp/development/ichiran.html
- **OpenStreetMap**（補助）

### 空間データ管理
- **PMTiles**（Protomaps形式・単一ファイル・静的ホスト可能）
  - 大量のベクタデータをVercel/Cloudflare R2にアップするだけで配信可
- 開発中は **GeoJSON**、本番は **PMTiles** に変換

### データ前処理
- **QGIS**: シェープファイル → GeoJSON → PMTiles のワークフロー
- **tippecanoe** (Mapbox製、PMTiles生成)
- **GDAL/ogr2ogr**: フォーマット変換
- **Python (GeoPandas)**: スクリプトでの一括処理

### バックエンド（必要になったら）
- 初期: 完全静的（フロント＋静的タイル）
- 拡張時: **Supabase (PostGIS)** または **Neon + PostGIS**
- API: Next.js API Routes / Hono

### ホスティング
- フロント: **Vercel**
- 大容量タイル: **Cloudflare R2**（エグレス無料）

## ユーザーの方針メモ
- **マルチテナント・横展開前提**は絶対（1自治体限定の実装は禁止）
- **地図UXが最重要差別化軸**（競合はここが弱い）
- **タブレット現場利用**を最優先（マウス前提のUIは却下）
- 伊丹市は最初のテナントにすぎない
- 委託業者が「明日から使える」タッチターゲット・視認性を最優先
- 側溝・下水道のような Google Maps にない情報の正確性は変わらず武器
- QGIS活用OK
