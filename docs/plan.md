# プロジェクト計画書 — Civic Map Ops（旧: 伊丹市自治体GISサービス）

## ⚠ 方針転換ログ（2026-04）
当初は「市民向け自治体GIS」として計画していたが、自治体予算化の道筋が弱いこと、
現場業務（側溝清掃・下水道点検）の紙/Excel運用に大きな非効率があることから、
**業務向け現場記録ツール（SaaS）** へ方針転換した。

旧計画（市民向けマップ・ゴミ出しナビ等）は `docs/plan_legacy_citizen.md`（※未作成）
として切り出す想定。モックの市民向けページは `mock/archive/citizen/` 以下に退避済み。

## ビジョン
「タブレット1台で、自治体インフラ維持管理の現場業務を完結させる。」
紙とExcelの世界を、地図とカードベースのUIで置き換える。

伊丹市を最初のテナントとして実績を作り、
全国の中規模都市（人口数万〜数十万）へマルチテナントSaaSとして横展開する。

## プロダクト
### 名称
- プロダクト名: **Civic Map Ops**（汎用ブランド）
- テナント表示例: 「伊丹市 環境インフラ台帳 / 伊丹市環境クリーンセンター」
- ヘッダーはテナント名を差し替える想定（data属性 or テナント設定で切替）

### ターゲットユーザー
#### Primary（現場オペレーター）
- 委託業者のスタッフ（側溝清掃・点検・補修の受託者）
- 自治体職員（現場に出る担当者）
- 用途: タブレットを持ち歩いて、現場で写真＋チェック＋メモ＋GPS記録

#### Secondary（管理者）
- 自治体職員（デスクで案件配分・進捗確認）
- 用途: どの案件が・どこに・どの状態か俯瞰し、優先度付け

#### Tertiary（SaaS購買決裁者）
- デジタル戦略課・環境クリーンセンター・上下水道局の管理職
- 購買材料: 紙/Excel運用からの工数削減量・事故時の証跡性

### 主要機能
1. **地図UI（閲覧モード）**: 案件・インフラ・担当エリアを一覧
2. **地図UI（記録モード）**: タップした場所に新規案件をピン立て
3. **ダッシュボード**: 本日の担当案件・KPI・アラート・活動ログ
4. **案件一覧**: 状態別フィルタ・検索・担当者・期日
5. **現場記録フォーム**: 写真・チェックリスト・状態評価・メモ・測定・位置情報
6. **オフライン対応**: PWA + IndexedDB で電波のない現場でも記録継続
7. **テナント切替**: 同一コードベースで複数自治体に対応

## リスクと対策

| リスク | 影響 | 対策 |
|------|-----|-----|
| 自治体の稟議が長い | 高 | 委託業者から攻めて実績を作り、業者経由で自治体に波及 |
| 既存システムとのデータ整合 | 中 | CSVエクスポートで既存台帳への橋渡しを確保 |
| 個人情報（住民情報）取扱 | 中 | 個人情報を持たない設計（案件情報のみ） |
| オフライン同期のデータ競合 | 中 | 案件は作業者単位で排他所有、サーバー側で最終書き込み勝ち |
| タブレット端末調達 | 低 | 持参BYOD対応、iPad/Android両対応 |
| マルチテナント設計ミス | 高 | Phase 2 時点からテナントIDで分離、後から後付けしない |
